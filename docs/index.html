📋 Overview

This is a complete, production-ready backend system built with NestJS, MongoDB, GraphQL, and JWT. It includes secure authentication with OTP email verification, role-based access control, and comprehensive financial records management with analytics dashboard.

Key Features

  • 🔒 Secure password hashing with bcrypt (12 rounds)
  • 📧 Email verification with 6-digit OTP
  • 🎯 JWT-based authentication (15-minute expiry)
  • 👥 Role-based access control (VIEWER/ANALYST/ADMIN)
  • 💰 Complete Financial Records Management (CRUD)
  • 📊 Dashboard Analytics with MongoDB Aggregation Pipeline
  • 🔐 Multi-layer Authorization (Role + Data Ownership)
  • 📱 GraphQL API with type-safe schema
  • 🛡️ Professional email templates
  • ⚡ Built with TypeScript for type safety

📡 Complete GraphQL API Reference

Copy these mutations and queries directly into GraphQL Playground or Postman

🔓 AUTHENTICATION APIs (Public)

1. User Signup

Purpose: Register new user | Auth: None required

mutation Signup { signup(signupInput: { email: "test@example.com" password: "password123" }) { token user { id email role isVerified } } }

2. Verify OTP

Purpose: Verify email address | Auth: None required

mutation VerifyOtp { verifyOtp(verifyOtpInput: { email: "test@example.com" otp: "123456" }) { success message } }

3. Resend OTP

Purpose: Get new OTP | Auth: None required

mutation ResendOtp { resendOtp(resendOtpInput: { email: "test@example.com" }) }

4. User Login

Purpose: Authenticate user | Auth: Account must be verified

mutation Login { login(loginInput: { email: "test@example.com" password: "password123" }) { token user { id email role isVerified } } }

5. Get Current User

Purpose: Fetch user profile | Auth: Bearer token required

query Me { me { id email role isVerified createdAt updatedAt } }

💰 FINANCIAL RECORDS APIs (Auth Required)

6. Create Financial Record

Purpose: Add income or expense | Auth: ANALYST/ADMIN only

mutation CreateFinance { createFinance(input: { amount: 5000 type: INCOME category: SALARY date: "2026-03-15" description: "Monthly salary payment" }) { _id userId amount type category date description createdAt updatedAt } }

Categories: SALARY | FOOD | UTILITIES | TRANSPORTATION | ENTERTAINMENT | HEALTHCARE | OTHER

7. List Financial Records

Purpose: Get records with filters | Auth: All roles | Data: Own records only

query ListFinances { listFinances( filter: { type: EXPENSE category: FOOD dateFrom: "2026-03-01" dateTo: "2026-03-31" } pagination: { skip: 0, take: 10 } ) { items { _id amount type category date description createdAt } total } }

8. Get Single Record

Purpose: Fetch record by ID | Auth: All roles

query GetFinance { getFinance(id: "RECORD_ID_HERE") { _id userId amount type category date description createdAt updatedAt } }

9. Update Financial Record

Purpose: Update amount/category/date | Auth: ANALYST/ADMIN | Data: Own records only

mutation UpdateFinance { updateFinance( id: "RECORD_ID_HERE" input: { amount: 6000 category: FOOD description: "Updated grocery expense" } ) { _id amount category description updatedAt } }

10. Delete Financial Record

Purpose: Delete record permanently | Auth: ANALYST/ADMIN | Data: Own records only

mutation DeleteFinance { deleteFinance(id: "RECORD_ID_HERE") }

📊 DASHBOARD ANALYTICS APIs (Auth Required)

11. Get Complete Dashboard

Purpose: All metrics for date range | Auth: All roles | Default: Current month

query GetDashboardSummary { getDashboardSummary( startDate: "2026-03-01" endDate: "2026-03-31" ) { totalIncome totalExpense netBalance categoryWiseTotals { category total count } recentActivity { _id amount type category date description } periodStart periodEnd } }

12. Get Total Income

Purpose: Sum of INCOME records | Auth: All roles

query GetTotalIncome { getTotalIncome( startDate: "2026-03-01" endDate: "2026-03-31" ) }

13. Get Total Expense

Purpose: Sum of EXPENSE records | Auth: All roles

query GetTotalExpense { getTotalExpense( startDate: "2026-03-01" endDate: "2026-03-31" ) }

14. Get Net Balance

Purpose: Income - Expense | Auth: All roles

query GetNetBalance { getNetBalance( startDate: "2026-03-01" endDate: "2026-03-31" ) }

15. Get Category Breakdown

Purpose: Total by category | Auth: All roles

query GetCategoryWiseTotals { getCategoryWiseTotals( startDate: "2026-03-01" endDate: "2026-03-31" ) { category total count } }

16. Get Recent Activity

Purpose: Last N transactions | Auth: All roles | Default: 10 records

query GetRecentActivity { getRecentActivity(limit: 10) { _id amount type category date description createdAt } }

📋 How to Use These APIs

GraphQL Playground:

1. Open http://localhost:3001/graphql

2. For protected queries, add header: {"Authorization": "Bearer YOUR_TOKEN"}

3. Copy query/mutation to left panel

4. Click Play button

Postman:

1. Set method to POST

2. URL: http://localhost:3001/graphql

3. Headers: {"Content-Type": "application/json", "Authorization": "Bearer token"}

4. Body: {"query": "YOUR_QUERY_HERE"}

🛡️ Security Features

🔒 Password Security

Passwords hashed with bcrypt (12 salt rounds) - computationally expensive to crack

🎯 JWT Authentication

JSON Web Tokens with 15-minute expiry. Contains userId for context extraction

📧 Email Verification

Mandatory email verification with 6-digit OTP codes (10-minute expiry)

👥 Role-Based Access

VIEWER (read-only) / ANALYST (own records) / ADMIN (all records)

🔐 Data Ownership

Users can only access/modify their own records (enforced at service level)

✅ Input Validation

All inputs validated with class-validator decorators (enum, format, positive)

🧹 Automatic Cleanup

Hourly cron job removes expired OTPs and tokens

Database Schema

User Collection: { _id: ObjectId, email: string (unique), password: string (bcrypt hashed), role: "VIEWER" | "ANALYST" | "ADMIN", isVerified: boolean, otp?: string, otpExpiry?: Date, createdAt: Date, updatedAt: Date } Finance Collection: { _id: ObjectId, userId: ObjectId (indexed), amount: number (positive), type: "INCOME" | "EXPENSE", category: string, date: Date (indexed), description?: string, createdAt: Date, updatedAt: Date }

⚙️ Environment & Setup

Required Environment Variables

# Database MONGO_URI=mongodb://localhost:27017/finance-db # JWT Configuration JWT_SECRET=your-super-secure-jwt-secret-key-here JWT_EXPIRY=15m # Email Configuration SMTP_HOST=smtp.gmail.com SMTP_PORT=587 SMTP_USER=your-email@gmail.com SMTP_PASSWORD=your-app-specific-password SENDER_EMAIL=noreply@yourapp.com

Installation & Running

# Install dependencies npm install # Copy environment variables cp .env.example .env # Edit .env with your configuration # Start development server npm run start:dev # Server runs on http://localhost:3001 # GraphQL Playground: http://localhost:3001/graphql # Production build npm run build npm run start:prod
